# 模块一工作日志：代码库扫描与项目管理

## 基本信息

- 开发日期：2026 年 7 月 17 日
- 开发分支：`feature/repository`
- 负责模块：代码库扫描与项目管理
- 对外接口：`POST /api/projects/load`

## 工作目标

接收本地项目路径，递归读取项目中的有效代码和文档文件，过滤依赖目录、缓存目录、二进制文件及超大文件，并将扫描结果提供给后续文本切分和索引模块。

## 已完成内容

1. 实现项目路径校验，能够识别路径不存在、路径不是目录和无访问权限等情况。
2. 实现递归文件扫描，默认支持以下文件类型：
   - `.py`
   - `.java`
   - `.js`
   - `.ts`
   - `.vue`
   - `.go`
   - `.cpp`
   - `.c`
   - `.md`
   - `.json`
   - `.yaml`
3. 实现目录过滤，默认忽略 `.git`、`.idea`、`node_modules`、`dist`、`build`、`.venv`、`venv`、`target`、`__pycache__` 及常见工具缓存目录。
4. 实现文件过滤，跳过符号链接、不支持的文件类型、二进制文件和超过 1 MiB 的文件。
5. 支持 UTF-8、UTF-8 BOM 和 GB18030 文本编码，无法解码时返回明确错误。
6. 定义统一文件数据结构，返回项目相对路径、文件类型、文件大小和文件内容。
7. 实现内存项目注册服务，供后续切分和索引模块按 `project_id` 获取完整文件内容。
8. 实现 `POST /api/projects/load` 接口，成功时返回项目 ID、项目路径、文件数量和文件元数据。
9. 为扫描器、项目服务和 API 编写模块说明及自动化测试。

## 主要文件

- `backend/repository/scanner.py`：路径校验、目录遍历、文件过滤与文本读取。
- `backend/repository/models.py`：扫描文件和扫描结果数据结构。
- `backend/repository/exceptions.py`：模块异常类型。
- `backend/repository/service.py`：已加载项目的内存注册服务。
- `backend/api/project_api.py`：项目加载 HTTP 接口。
- `tests/test_repository.py`：扫描器和项目服务测试。
- `tests/test_project_api.py`：项目加载 API 测试。

## 开发中遇到的问题

1. Windows 测试环境的系统临时目录存在访问限制。测试改用项目目录下的独立临时测试目录，并在测试结束后自动清理。
2. API 返回值只暴露前端需要的文件元数据，完整文件内容保存在项目服务中，避免接口响应过大，同时满足后续模块读取内容的需求。
3. 为保证跨平台路径一致，所有文件路径统一转换为以 `/` 分隔的项目相对路径。
